WIRES Website Redesign Project

What would you do if you find an injured native animal?

Our group of 4 UX/UI Designers, were wondering the same & we were pleased to discover a Not-for-profit organisation called WIRES, providing wildlife rescue services.

Project Overview

Role

UX/UI Designer

Time

3 weeks

Tools

Figma, Miro, Google Drive

Challenge

Design a new website for WIRES & improve on the existing navigation system for a more seamnless & untuative user experience.

Solution

  • Communicate WIRES’s main services/offerings on the homepage e.g. native animal rescue, fundraising, donate
  • Clear mission statement
  • Re-categorise the information architecture and clear labelling for ease of navigation
  • Address colour contrast
  • Procedure outlining next steps for reporting a rescue
  • Add latest events & updates to homepage
  • Create responsive design to ensure that information does not get lost or hidden when users are viewing the site on different devices

Deliverables

  • User Interviews
  • Heuristic Evaluation
  • Problem Statement
  • Proto Persona
  • Storyboard
  • Competitor Analysis
  • Feature Prioritisation Matrix
  • Card Sorting & Navigation System
  • User Flow
  • Style Guide
  • User Testing & Iterations
  • Hi-Fi Wireframes Desktop & Mobile

Process

The following Design Process was applied throughout our Project:

  • Research  
  • → Define
  • → Ideate
  • → Prototype
  • → Test

Research

Since we were not familiar with the existing WIRES website we began with a Heuristic Evaluation of the existing elements.

Heuristic Evaluation

At first glance the homepage looks to be clearly laid out with good content grouping. The navigation is simple but key features were hidden in the burger menu.

Another major problem with the site was that the lime green and orange colours failed WCAG AAA and AA accessibility standards.

Overall, users commented that the styling looked a little amateurish and felt a bit clinical.

User Testing

We then conducted 10 user interviews to learn more about the different experiences people had navigating the site and to identify potential pain points.

Through our reserach, we wanted to find out:


  • how users are navigating the website & see if they can find the information they need
  • are the users satisfied using the website
  • can the users make a donation easily
  • pain points the user encounters & learn what features work well and what can be improved on.

Here are some insights from the interviews:

Difficult to identify WIRES’s Mission Statement & Values

Main services & offerings not highlighted enough

There was too much text, confusing wording & long forms

Search option was not effective

Labelling of buttons inconsistent

Lack alerting when leaving main site

Competitor Analysis

Next, we conducted a competitor analysis of similar wildlife rescue and information website based on 4 main factors:

  • Color scheme & accessibility
  • Text readability and Information layout
  • Navigation and Responsive design

Some of our key findings were:

  • simple information architecture and clear labelling is important for ease of navigation
  • responsive design is important to ensure that information does not get lost or hidden users are viewing the site on different devices.

Define

Next we defined a typical WIRES user.

Proto Persona

Meet Trudy, an Environmental Science uni student. She has seen the affects that bushfires have had on native animals in her local area and wants to learn more about what she can do to help.

To demonstrate how Trudy would interact with the website, we created a User Journey in which Trudy found an injured possum in the wild.

Problem Statement

By analysing and synthesising our collected data, we asked ourselves the following question:

How can we make it clearer & easier for users to find the relevant information on rescue animals?

Ideate

We then created an Affinity Diagram of the user test results to group the data into common themes. Further analysis utilizing a Frequency Diagram was conducted to establish the core issues.

Feature Prioritisation Matrix

A 2x2 Feature Prioritisation Matrix was conducted to establish what aspects of the website should be addressed.

Priority features to be addressed were to:


  • communicate the main services/offerings from WIRES on the homepage i.e. native animal rescue, fundraising & donat
  • display a clear mission statement
  • re-categorise the information architecture with clear labelling
  • address colour contrast
  • procedure outlining next steps for reporting a rescue
  • add latest events to homepage.

Prototype

Then we moved on to the prototype phase of our project in which we also addressed the existing information architecture of WIRES.

Card Sorting

To the address navigational issues, card sorting was utilized to establish new groupings, labels and structure.

The results were then used as the new primary and secondary navigation system for the websites header & a full version collated for the website footer.

Low Fidelity Wireframes

Next, we created some Low/Mid Fidelity Wireframes to determine the new structure of the site as well as testing out different colour combinations before determining our colour schemes and other UI elements. We also conducted some A/B testing to gain more insights as to which style others prefer.

UI Style Guide

Afterwards, a UI Style Guide was developed to solidify important UI elements of the website, including logo measurements, colour scheme & typography.

Based on our Colour Accessibility results, we decided to change the colour scheme to a more earthy green & orange, while still adhering to accessibility guidelines by increasing the contrast between certain colour combinations.

Logo

Colour

We then applied the Style Guide to create the first iteration of our High Fidelity Wireframes.

Some of the key changes were:

  • A simpler header navigation with 4 tabs instead of 10
  • More spacing & images between paragraphs to increase readability
  • Footer navigation with links to all pages on the site

Responsive Design

To demonstrate the responsive design of the desktop, mobile wireframes with the applied Style Guide were created.

Test

Following the development of our High Fidelity Prototype, we conducted user testing that included similar tasks as the initial tests performed on the WIRES website. We did this intentionally to gauge, whether we had improved the navigation, labeling & communication, as well as other issues previously identified.

Key findings:

  • Users could quickly find important information such as the emergency rescue number & report a rescue form
  • It was a simple process to make a donation
  • Some users did not instinctively scroll down to find more information
  • Some headings in the dropdown menu were still ambiguous & it was not in every case easy to find the bushfires factsheet

Design Iterations

Through 7 user tests, we implemented their feedback into the next round of iterations.
Here, you can see the before and after screenshot of the Original WIRES website and our re-design version.

Further User Testing

After testing out this iteration with more users, we found that it was much easier for users to:

  • locate the bushfire guidelines
  • identify more content on the homepage

Some users did mention that they would still prefer to search for information, which leads us to consider the next steps for future iterations.

Final Thoughts & Opportunities

If we had more time, we would like to develop branded training material for the volunteering section & design some e-newsletter templates.
We also felt that the search function needed improving as our initial analysis showed that the search results were unreliable.

As for future opportunities, we feel that creating a members hub, where members could see their personalised events & volunteer schedule would make users feel more involved and engaged with WIRES.

Next steps:
