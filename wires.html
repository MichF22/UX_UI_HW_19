WIRES Website Redesign Project

What would you do if you find an injured native animal?

Our group of 4 UX/UI Designers, were wondering the same & we were pleased to discover a Not-for-profit organisation called WIRES, providing wildlife rescue services.

Process

The following Design Process was applied throughout our Project:

  • Research 
  • • Define
  • • Ideate
  • • Prototype
  • • Test

Research

Since we were not familiar with the existing WIRES website we began with a Heuristic Evaluation of the existing elements.

Heuristic Evaluation

At first glance the homepage looks to be clearly laid out with good content grouping. The navigation is simple but key features were hidden in the burger menu.

Another major problem with the site was that the lime green and orange colours failed WCAG AAA and AA accessibility standards.

Overall users commented that the styling looked a little amateurish and felt a bit clinical.

User Testing

We then conducted 10 user interviews to learn more about the different experiences people had navigating the site and to identify potential pain points.

Through our reserach, we wanted to find out:

  • how users are navigating the website & see if they can find the information they need
  • are the users satisfied using the website
  • can the users make a donation easily
  • pain points the user encounters & learn what features work well and what can be improved on.

Here are some insights from the interviews:

Difficult to identify WIRES’s Mission Statement & Values

Main services & offerings not highlighted enough

There was too much text, confusing wording & long forms

Search option was not effective

Labelling of buttons inconsistent

Lack alerting when leaving main site

Competitor Analysis

Next, we conducted a Competitor Analysis of similar wildlife rescue and information website based on 4 main factors: Color scheme & accessibility, Text readability and Information layout, Navigation and Responsive design.

Some of our key findings were:

  • Simple information architecture and clear labelling is important for ease of navigation
  • Responsive design is important to ensure that information does not get lost or hidden users are viewing the site on different devices

Define

Next we defined a typical WIRES user outlined below.

Proto Persona

This is Trudy, she’s a uni student studying Environmental Science. She has seen the affects that bushfires have had on native animals in her local area and wants to learn more about what she can do to help.

To demonstrate how Trudy would interact with the website, we created a User Journey of a scenario where Trudy found an injured possum in the wild.

User Testing

By analysing and synthesising our collected data, we asked ourselves the following question:

How can we make it clearer & easier for users to find the relevant information on rescue animals?

Ideate

Next we created an Affinity Diagram of the user test results to group the data into common themes. Further analysis utilizing a Frequency Diagram was conducted to establish the core issues.

Feature Prioritisation Matrix

A 2x2 Feature Prioritisation Matrix was conducted to establish what aspects of the website should be addressed in this case study.

Priority features to be addressed:

Communicate the main services/offerings from WIRES on the Homepage. These being - Native Animal Rescue, Fundraising, Donat

Clear mission statement

Re-categorise the information architecture with clear labelling

Address colour contrast

Procedure outlining next steps for reporting a rescue

Add latest events to homepage

Prototype

Then we moved on to the prototype phase of our project in which we also addressed the existing information architecture of WIRES.

Card Sorting

To the address navigational issues, card sorting was utilized to establish new groupings, labels and structure.The results were then used as the new primary and secondary navigation system for the websites header and a full version collated for the website footer.

Lo-Fi Wireframes

Next, we created some low/mid-fidelity wireframes to determine the new structure of the site as well as testing out different color combinations before determining our color schemes and other UI elements. We also conducted some A/B testing to gain more insights as to which style others prefer.

UI Style Guide

Afterwards, a UI Style Guide was developed to solidify important UI elements of the website, including logo measurements, color scheme and typography. Based on our Color Accessibility results, we’ve decided to change the color scheme to a more earthy green and orange, while still adhering to accessibility guidelines by increasing the contrast between certain color combinations.

We then applied the Style Guide to create the first iteration of our High Fidelity Wireframe.

Some of the key changes were:

A simpler header navigation with 4 tabs instead of 10

More spacing and images between paragraphs to increase readability

Footer navigation with links to all pages in the site

Responsive Design

To demonstrate the responsive design of the desktop to mobile wireframes were created and the style guide applied.

Test

Following the development of our high fidelity prototype, we conducted user testing that included similar tasks as the initial tests performed on the WIRES website. We did this intentionally to gauge Whether we had improved the navigation, labeling and communication As well as other issues previously identified.

Key findings:

Users could quickly find important information such as the Emergency Rescue Number and Report a Rescue Form

It was a simple process to make a donation

Some users did not instinctively scroll down to find more information

Some headings in the dropdown menu were still ambiguous and it was not in every case easy to find the bushfires factsheet.
